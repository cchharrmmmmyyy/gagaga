# Memory Game & Flappy Bird 改进计划

## 一、项目现状分析

### Memory Game
- **技术栈**：纯原生 JS + CSS，无外部依赖
- **当前状态**：基本可玩，但界面简陋，无分数统计、无计时、无游戏状态控制
- **核心逻辑**：16 张卡片（A-H 各两张），点击翻牌配对，匹配后禁用，不匹配 1.5 秒后翻回
- **缺失功能**：计分、计时、游戏结束判定、开始/暂停、提示、规则说明、主题切换

### Flappy Bird
- **技术栈**：Canvas 原生绘制
- **当前状态**：存在严重可玩性问题
- **核心逻辑**：按任意键上升，受重力下落，穿过绿色管道，碰壁或碰管则 Game Over
- **主要问题**：
  1. **操控性极差**：`lift = -5`，`gravity = 0.25`，下落太快，按一下飞太高，手感僵硬
  2. **管道间距太小**：`pipeGap = 100` 在 480px 高的画布上过窄，几乎无法穿过
  3. **无分数显示**：没有计分模块
  4. **无开始/暂停**：页面加载即开始，Game Over 直接 reload
  5. **画面粗糙**：鸟是黄色方块，管道是绿色方块，无任何装饰
  6. **碰撞检测不精确**：使用矩形碰撞，鸟是 20x20 方块，视觉和手感都不好

## 二、分阶段改进计划

我们将分 **5 个阶段** 逐步完善，每个阶段完成后测试验证，再进入下一阶段。

---

### 阶段一：UI 美化与主题系统

**目标**：为两个游戏页面增加视觉吸引力，实现暗/亮双主题切换。

#### 1.1 全局主题系统
- 在项目根目录或每个游戏页面引入 CSS 变量（`:root` 和 `[data-theme="dark"]`）
- 提供主题切换开关（按钮），用户可切换暗/亮主题
- 主题设置保存到 `localStorage`，刷新后保持

#### 1.2 Memory Game 美化
- 卡片样式升级：圆角、阴影、翻转动画（CSS 3D transform）
- 卡片内容用 emoji 或图标替代纯字母（如 🍎🍊🍋🍇🍉🍓🍑🍒）
- 背景渐变色，卡片排列更美观
- 匹配成功/失败的动画效果

#### 1.3 Flappy Bird 美化
- 鸟用圆形 + 眼睛（Canvas 绘制）替代矩形
- 管道增加细节（管道口装饰）
- 背景增加云朵、地面等装饰元素
- 分数显示在画布上

**涉及文件**：
- `Memory Game/index.html` — 引入新 CSS，增加主题切换
- `Memory Game/memory.js` — 适配新样式，增加翻转动画控制
- `Flappy Bird/index.html` — 引入新 CSS，增加主题切换
- `Flappy Bird/flappybird.js` — Canvas 绘制优化

---

### 阶段二：游戏增强模块

**目标**：为两个游戏增加完整的游戏体验模块。

#### 2.1 规则讲解模块
- Memory Game：显示游戏规则（翻牌配对，全部匹配即获胜）
- Flappy Bird：显示操作说明（点击/按空格上升，穿过管道得分）
- 实现方式：可折叠面板 / 模态框 / 侧边提示

#### 2.2 游戏提示系统
- Memory Game：可消耗一次机会显示所有未匹配卡片（短暂闪烁）
- Flappy Bird：游戏开始前显示操作提示

#### 2.3 分数显示模块
- Memory Game：显示已匹配对数 / 总对数（如 2/8）
- Flappy Bird：显示当前得分（穿过管道数）、最高分（localStorage 保存）

#### 2.4 时间计时器
- Memory Game：记录游戏用时（mm:ss 格式），游戏结束时显示总用时
- Flappy Bird：可选，记录单局存活时间

#### 2.5 游戏状态控制
- 开始画面：显示标题、规则、开始按钮
- 暂停功能：游戏过程中可暂停（空格 / 按钮）
- 游戏结束：显示结果（分数、用时）、重新开始按钮
- 避免 Memory Game 中 `alert` 的使用，改用页面内提示

**涉及文件**：
- `Memory Game/memory.js` — 重写游戏循环，增加状态机
- `Flappy Bird/flappybird.js` — 重写游戏循环，增加状态机
- 两个游戏的 HTML/CSS 同步更新

### 阶段三：Flappy Bird 核心体验优化

**目标**：让 Flappy Bird 变得可玩、好玩。

#### 3.1 物理参数调优
- 降低重力：`gravity` 从 0.25 → 0.15
- 调整升力：`lift` 从 -5 → -4（更轻柔的上升）
- 增加上升角度限制，防止连续快速按导致飞太高
- 增加下落最大速度限制（terminal velocity）

#### 3.2 管道参数优化
- 增大管道间距：`pipeGap` 从 100 → 130~150（根据画布高度动态计算）
- 降低管道生成频率：`frame % 90` → `frame % 110~120`
- 管道速度微调：`pipeSpeed` 从 2 → 1.8
- 管道高度生成更合理：避免出现极端高/低的情况

#### 3.3 操控优化
- 支持鼠标点击 + 空格键 + 触屏
- 增加点击防抖，防止误触
- 每次点击强制设置速度为 lift（而非累加），手感更一致
- 鸟的旋转动画：上升时抬头，下落时低头

#### 3.4 碰撞检测优化
- 圆形碰撞检测替代矩形碰撞（鸟是圆形，管道保持矩形）
- 或者使用更宽松的矩形碰撞（缩小碰撞盒）
- 碰撞后增加死亡动画（而非立即 reload）

#### 3.5 其他体验优化
- 得分反馈：穿过管道时有视觉/文字反馈
- 地面滚动效果
- 难度渐进：随着分数增加，管道速度略微加快

**涉及文件**：
- `Flappy Bird/flappybird.js` — 大量重写

### 阶段四：Memory Game 体验增强

**目标**：让 Memory Game 更完整、更有挑战性。

#### 4.1 游戏结束判定
- 所有卡片匹配后显示胜利画面
- 显示总用时和步数（点击次数）
- 显示最佳记录（localStorage）

#### 4.2 难度选择
- 简单模式：4×4（8对，当前）
- 困难模式：4×6（12对，新增卡片 I-L）
- 通过按钮切换

#### 4.3 动画与音效
- 卡片翻转 3D 动画
- 匹配成功：绿色高亮 + 缩放动画
- 匹配失败：红色闪烁
- 游戏结束：庆祝动画（彩纸或烟花效果）

#### 4.4 操作反馈
- 点击卡片时增加按下效果
- 匹配成功/失败的声音提示（用 Web Audio API 生成简单音效，无需外部文件）

**涉及文件**：
- `Memory Game/memory.js` — 重写
- `Memory Game/index.html` — 增加难度选择等 UI

---

### 阶段五：代码重构与优化

**目标**：代码整洁、可维护、可扩展。

#### 5.1 代码架构
- 将游戏逻辑封装为类（Class）或模块
- 状态机管理：`IDLE` → `PLAYING` → `PAUSED` → `GAME_OVER`
- 分离 UI 渲染和游戏逻辑

#### 5.2 性能优化
- requestAnimationFrame 替代 setInterval（Flappy Bird）
- Canvas 离屏渲染（如需）
- 事件监听优化，避免内存泄漏

#### 5.3 响应式适配
- 画布尺寸自适应屏幕
- 移动端触屏支持优化

## 四、技术要点备忘

### 主题切换实现思路
```css
:root {
  --bg: #ffffff;
  --text: #333333;
  --card-bg: #cccccc;
}
[data-theme="dark"] {
  --bg: #1a1a2e;
  --text: #e0e0e0;
  --card-bg: #16213e;
}
```
用 JS 切换 `document.documentElement.dataset.theme`，并保存到 `localStorage`。

### 游戏状态机
```
         ┌──────────┐
         │   IDLE   │  ← 初始/开始画面
         └────┬─────┘
              │ 点击开始
         ┌────▼─────┐
         │ PLAYING  │  ← 游戏进行中
         └────┬─────┘
        ┌─────┴──────┐
        │            │
   ┌────▼────┐  ┌────▼──────┐
   │ PAUSED  │  │ GAME_OVER │
   └────┬────┘  └────┬──────┘
        │            │
        └─────┬──────┘
              │ 重新开始
         ┌────▼─────┐
         │   IDLE   │
         └──────────┘
```

### Flappy Bird 物理参数调优建议
- `gravity`: 0.15（现 0.25）
- `lift`: -4（现 -5）
- `pipeGap`: 画布高度 × 0.3 ≈ 144（现 100）
- `pipeSpeed`: 1.8（现 2）
- 管道生成间隔: 110~120 帧（现 90 帧）
- 最大下落速度: 6（新增）
- 每次点击重置速度为 lift，不累加
